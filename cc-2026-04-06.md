# Session Summary — 2026-04-06

## Fixes

### "Rate-limited — retry in now" (root cause found)
- `_parse_retry_after` was returning `0.0` because the API sends `Retry-After: 0`
- Fixed: treat `api_wait <= 0` as `None` → fall through to exponential backoff (starts at 5m)
- Also fixed `_format_relative`: check `seconds <= 0` (float) before truncating to int, use `max(1, int(seconds))` so sub-1s values show as "1s" not "now"

### Cursor drift — content scrolling up with each redraw
- `_clear_lines(n)` was doing `\033[1A\033[2K` n times, but cursor was mid-line (end of last printed line with `end=""`)
- Each redraw moved content one line higher, eating terminal history
- Fixed: start with `\r\033[2K` (clear current line in-place), then `\033[1A\033[2K` only `n-1` times

### Extra blank line above output
- Startup `print()` + `_clear_lines(1)` was intended as a scroll buffer for the old up-first approach
- With new in-place clearing, the blank line was no longer consumed — just left on screen
- Fixed: removed both `print()` and `_clear_lines(1)` from `main()`
